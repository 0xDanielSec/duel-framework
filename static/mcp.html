run_battle
run_battle(technique_id, rounds=5, logs_per_round=10) → dict
Run a full adversarial battle. The Attacker generates synthetic Sentinel telemetry; the Defender generates KQL detection rules. Each round the Attacker mutates based on what was caught.
technique_id str — MITRE ID (T1078.004) or OWASP LLM ID (LLM01)

HOW TO CONNECT
  1. Install Claude Desktop if you haven't already.
  2. Open claude_desktop_config.json — location: ~/Library/Application Support/Claude/ (macOS) or %APPDATA%\Claude\ (Windows).
  3. Add the DUEL server block shown below, updating the path to match your clone location.
  4. Restart Claude Desktop. The DUEL tools will appear in the tool picker.

claude_desktop_config.json
{
  "mcpServers": {
    "duel": {
      "command": "python",
      "args": ["C:/Users/YOU/duel-framework/mcp_server.py"],
      "env": {}
    }
  }
}
HOW TO CONNECT
  1. Open Cursor Settings → MCP section (or edit .cursor/mcp.json in your project root).
  2. Add the server config below, updating the path to your clone.
  3. Reload Cursor. DUEL tools will be available in Agent mode.

.cursor/mcp.json
{
  "mcpServers": {
    "duel": {
      "command": "python",
      "args": ["C:/Users/YOU/duel-framework/mcp_server.py"],
      "env": {}
    }
  }
}
TEST THE SERVER DIRECTLY
  1. Install dependencies: pip install mcp
  2. Start the server: python mcp_server.py
  3. The server listens on stdio. Use the MCP Inspector or send JSON-RPC 2.0 messages directly.
  4. Or use: npx @modelcontextprotocol/inspector python mcp_server.py
